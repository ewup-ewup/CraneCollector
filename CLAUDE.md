# Crane Collector - 개발 가이드

## 프로젝트 정보
- **프로젝트명**: Crane Collector (Web-based 3D Claw Machine)
- **기술 스택**: Three.js, Cannon-es, HTML5
- **목표**: 도파민 중심 인형뽑기 시뮬레이터 → 모바일 앱 출시 → 광고/인앱결제 수익화

---

## 핵심 메커니즘

### 1. 橋渡し(다리걸기) 인형뽑기
- 두 평행한 봉 위에 박스가 가로로 걸쳐있음
- 집게의 약한 힘으로 직접 들어올리기 거의 불가능
- 무게중심 이동/회전/사이드 어택으로 봉 사이로 떨어뜨림
- 봉 사이를 통과한 박스만 획득

### 2. 박스 등급 공개 (이치방쿠지 방식)
- 모든 박스는 외관상 동일
- 획득 후 페이크 연출로 등급 공개
- 5등급: F < E < D < C < B < A < **Last One**

### 3. Last One 메커니즘
- 각 기계마다 박스 잔여 수 표시 (예: 15/30)
- 마지막 박스 = 자동 Last One 보너스

---

## 도파민 설계 (필수 구현)

- [x] 박스 획득 후 2단계 등급 공개 (스킬 + 운)
- [x] 페이크 연출 (보라색 → 파란색 → 금색 상승)
- [x] "거의 성공" 시각 피드백 (60% 실패 연출)
- [x] 잔여 박스 가시화
- [x] 시도 누적 시 확률 보정 (피티)

---

## 보상 시스템

### 광물/결정 (절차적 셰이더)
```
F: 회색 결정 (1색)
E: 단색 (청/녹/적)
D: 2색 그라데이션
C: 다색 + 내부 구조
B: 홀로그램 효과
A: 무지개 분광 + 동적 반사
Last One: 황금 + 무지개 + 파티클
```

### 메타 보상
- 트로피/칭호 (업적 시스템)
- 프로필 진열 가능

---

## 모바일 조작

- **메인**: 터치 드래그 (좌우/전후)
- **보조**: 화면 하단 버튼 (내림, 회전)
- **카메라**: 핀치 줌 + 정/측면 토글
- **피드백**: 진동 (성공/실패/Near Miss)

---

## 개발 단계

### Phase 1: 핵심 물리 (현재)
- Three.js 기본 씬
- Cannon-es 물리 엔진
- 크레인 + 박스 + 두 봉
- 기본 인터랙션 (터치 드래그)

### Phase 2: 게임 루프
- 획득 판정 로직
- 등급 공개 연출
- 점수/통계 추적

### Phase 3: UI/UX
- 메인 메뉴
- 게임 UI (남은 시간, 박스 잔여수)
- 결과 화면

### Phase 4: 보상 시스템
- 광물 셰이더
- 도감/수집
- 통계

### Phase 5: 데일리 루프 + 수익화
- 무료 뽑기 1회
- 광고 시청 (추가 뽑기)
- 인앱 코인 구매
